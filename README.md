# OmniCortex

OmniCortex is a multi-agent RAG backend with:
- FastAPI APIs for agent lifecycle, chat, documents, channels, tools, and webhooks.
- PostgreSQL + pgvector as source-of-truth runtime storage.
- vLLM/OpenAI-compatible inference backends.
- Optional ClickHouse analytics for usage and chat archive reporting.
- Moshi/PersonaPlex voice WebSocket proxy.

This README reflects the current implementation in this repo.

## Current Status (March 2026)

- Auth is bearer-token based and externally verified (`Authorization: Bearer <token>`).
- Agent `id` is client-provided in create payloads (not generated by server).
- `/query` uses `id` (agent id) in request payload.
- `model_selection` is configured on agent create/update and resolved from agent config at query time.
- Voice REST endpoints are disabled in Moshi-only mode; use `/voice/ws`.
- Canonical media tags are enforced in responses:
  - `[image][filename.ext]`
  - `[video][filename.ext]`
  - `[document][filename.ext]`

## Architecture

- API server: `api.py` (FastAPI)
- Core services: `core/`
  - agent CRUD: `core/agent_manager.py`
  - chat + RAG orchestration: `core/chat_service.py`
  - LLM invocation: `core/llm.py`
  - vector retrieval: `core/rag/`
  - auth callback validation: `core/auth.py`
  - analytics writer: `core/clickhouse.py`
- Prompts:
  - personal: `prompts/personal/*.json`
  - business: `prompts/business/*.json`

## Requirements

- Python 3.10+ (3.12 recommended)
- PostgreSQL with pgvector extension
- vLLM/OpenAI-compatible inference endpoint
- Optional: ClickHouse
- Optional: Moshi/PersonaPlex server for voice proxy
- Optional: Voice Gateway (`scripts/voice_gateway.py`) for FreeSWITCH/media WS bridging

## Auth Model

All protected APIs use:

```http
Authorization: Bearer <token>
```

Auth verification is delegated to external callback (`AUTH_VERIFY_URL`) in `core/auth.py`.

Optional request header:

```http
X-User-Id: 102
```

`X-User-Id` is:
- forwarded to auth verification callback
- persisted on agent create/update as `omni_agents.user_id`
- used as fallback user identity in query/session logging

## Agent Payload Contract

### Create Agent (`POST /agents`)

Important:
- `id` is required and is used as the persisted agent id.
- `name` can be supplied as `name`, `agent_name`, or `agentname`.
- `conversation_starters` accepts string or object entries.
- `conversation_end` accepts string or object entries.
- `logic` is stored in dedicated DB column (not in `extra_data`).
- `model_selection` is set here (or in update), not in `/query`.

Example:

```bash
curl -X POST "http://<api_host>:<api_port>/agents" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -H "X-User-Id: 102" \
  -d '{
    "id": "<agent_id>",
    "agentname": "Personal AI BOT",
    "agent_type": "PersonalAssistant",
    "subagent_type": "NULL",
    "role_type": "PersonalAssistant",
    "description": "For My Assignment Helping.",
    "model_selection": "Meta Llama 3.1",
    "website_data": [
      "https://www.w3schools.com/sql/",
      "https://www.w3schools.com/python/"
    ],
    "document_data": {
      "image_urls": [
        "https://i0.wp.com/learn.onemonth.com/wp-content/uploads/2019/07/image2-1.png",
        "https://kunalcybersecurity.com/wp-content/uploads/2023/08/Python-Symbol.png"
      ],
      "video_urls": [
        "http://commondatastorage.googleapis.com/gtv-videos-bucket/sample/TearsOfSteel.mp4"
      ],
      "documents_text": [
        {
          "url": "https://ncert.nic.in/textbook/pdf/keip103.pdf",
          "type": "pdf"
        }
      ]
    },
    "logic": {
      "handler": "personal-ai-bot",
      "timeout_seconds": "120",
      "max_tokens": "4096",
      "rule": ""
    },
    "conversation_starters": [
      { "prompt": "Hola Amigo!!!" }
    ],
    "conversation_end": [
      { "prompt": "Thank you for learning with Smart AI" }
    ]
  }'
```

Create response:

```json
{
  "status": "created",
  "id": "<agent_id>",
  "agent_name": "Personal AI BOT"
}
```

## Query Contract

### Query (`POST /query`)

Example:

```bash
curl -X POST "http://<api_host>:<api_port>/query" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -H "X-User-Id: 102" \
  -d '{
    "id": "<agent_id>",
    "channel_name": "TEXT",
    "channel_type": "MARKETING",
    "query": "Send me course images"
  }'
```

Response:

```json
{
  "answer": "...",
  "id": "<agent_id>",
  "session_id": "<session_id>",
  "request_id": "<request_id>"
}
```

Behavior notes:
- `id` is the agent id.
- If `session_id` is not passed, API auto-creates/reuses a daily session per `(agent_id, user_id, channel_name)`.
- `user_id` resolution priority:
  1. body `user_id` (if not empty and not `anonymous`)
  2. header `X-User-Id`
  3. `anonymous`
- `model_selection` is resolved from agent config.

## Rich Media Tags

LLM is instructed to use canonical tags:
- `[image][filename.jpg]`
- `[video][filename.mp4]`
- `[document][filename.pdf]`
- `[link][url][text]`
- `[location][lat,long][name][address]`
- `[buttons][Title][Option1|Option2|Option3]`

Pipeline:
- `core.chat_service.process_question` enforces canonical media tags.
- `core.response_parser` converts tags to:
  - frontend markdown
  - structured parts for WhatsApp dispatch

## URL Scraping and File Ingestion

When creating/updating agent:
- `urls` can contain web pages and supported direct file links (`pdf`, `txt`, `csv`, `docx`).
- `core/processing/scraper.py`:
  - scrapes HTML pages
  - downloads supported files (including GitHub blob URL normalization to raw URLs)
  - ingests content via document pipeline
  - removes temporary downloads after ingestion

## Voice

Moshi-only mode is active:
- Use WebSocket: `WS /voice/ws`
- Disabled REST endpoints return 501:
  - `POST /voice/transcribe`
  - `POST /voice/speak`
  - `POST /voice/chat`
- Legacy Liquid endpoint returns 410:
  - `POST /api/v1/voice/liquid`

### Moshi Server (with OmniCortex UI patch)

```bash
cd /workspace/OmniCortex
source .moshi-venv/bin/activate

export OMNICORTEX_BASE_URL=http://127.0.0.1:8000
export OMNICORTEX_API_KEY=<bearer_token_for_agents_api>
export MOSHI_ENABLE_OMNICORTEX_UI_PATCH=1

python -m moshi.server \
  --host 0.0.0.0 \
  --port 8998 \
  --device cuda
```

### Voice Gateway (FreeSWITCH/media WS -> OmniCortex `/voice/ws`)

`scripts/voice_gateway.py` exposes a WS endpoint (default `/calls`) and bridges audio to OmniCortex voice proxy.

Run:

```bash
cd /workspace/OmniCortex
source .moshi-venv/bin/activate

python scripts/voice_gateway.py \
  --host 0.0.0.0 \
  --port 8099 \
  --endpoint /calls \
  --omnicortex-voice-ws ws://127.0.0.1:8000/voice/ws \
  --default-agent-id <agent_uuid> \
  --default-token <bearer_token> \
  --default-voice-prompt NATF0.pt \
  --inbound-mode pcm16 \
  --outbound-mode pcm16 \
  --fs-sample-rate 8000 \
  --moshi-sample-rate 24000
```

For direct WSS on gateway process:

```bash
python scripts/voice_gateway.py \
  --host 0.0.0.0 \
  --port 443 \
  --endpoint /calls \
  --ssl-cert /path/to/fullchain.pem \
  --ssl-key /path/to/privkey.pem \
  --omnicortex-voice-ws ws://127.0.0.1:8000/voice/ws
```

If TLS is terminated by nginx, run gateway on `8099` and proxy `/calls` with websocket upgrade headers.

Detailed guide: `docs/VOICE_GATEWAY.md`.

## Common Issues

- `403 Authorization Bearer token missing`
  - Missing or invalid `Authorization: Bearer <token>`.
- `403 Invalid bearer token`
  - Token failed external auth callback verification.
- `422` on `POST /agents`
  - Missing required `id` in payload.
- `500` during query with backend error
  - Check vLLM endpoint (`VLLM1_BASE_URL`) and loaded model.
- Voice REST returns `501` / `410`
  - Expected in Moshi-only mode; use `/voice/ws`.
- Moshi UI still shows stock `Examples`
  - Ensure `MOSHI_ENABLE_OMNICORTEX_UI_PATCH=1` and set `OMNICORTEX_API_KEY` or provide bearer token in UI.
- `FileNotFoundError` for TLS cert on Voice Gateway
  - Cert path is wrong/missing in container; use mounted certs or terminate TLS at nginx.
